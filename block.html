A very simple block

#


The Block is like a basic entity in a blockchain system.

  • Block(#): Indicates the index of the block.
  • Data: consists the body of the block which consists of any data required for the functioning of the blockchain system.
    Note:    A small change in the data in the block also leads to a large change in the hash value of the block.
  • Mine: this initiates a process of solving a complex mathematical guessing problem to make a block valid by adjusting the Nonce value.
  • Nonce: an integer value that can be varied independent of the data so that a solution can be obtained during mining.
  • Prev Hash: containts the previous hash value of the block in a blockchain.

    • A basic block usually has the above mentioned parameters in its structure.For a block to be valid in a blockchain system it needs to abide by system rules. In our example, the blockchain system rule is that the hash value should begin with four zeros. Our rules are relatively simple and easily attainable by a standalone system for demonstration purposes. The process of validating a block is called as mining and it is done by iterating through different nonce values till the required result that abides by the validation rule is obtained. Processing power is expended on solving a complex mathematical problem. The processor that successfully mines a block is rewarded in someway and the block gets added to the blockchain.
